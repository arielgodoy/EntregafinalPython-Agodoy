Métodos soportados
GET

PUT
DELETE
Autenticación

La API usa sesión autenticada del sistema (cookie sessionid). Para métodos POST/PUT/DELETE se requiere CSRF token. No requiere token externo manual.

Headers requeridos
Accept: application/json
X-Requested-With: XMLHttpRequest
X-CSRFToken: <csrftoken>
Contexto de empresa

El sistema es multiempresa: la empresa activa se obtiene desde la sesión del usuario. Esta consola consume exclusivamente endpoints internos del proyecto.

Resolución dinámica rubro/local
rubro y local son obligatorios.
base = eltit_gestion + rubro
tabla = l_movimientos_cabeza_ + local
Nota sobre la PK legacy

La PK real del legacy es tipo + numero + fecha. Esta API opera por tipo + numero y devuelve 409 si existe ambigüedad por múltiples fechas.
